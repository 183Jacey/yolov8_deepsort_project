# CPU与云GPU环境分工适配方案
## 一、核心原则
本地Ubuntu CPU：仅负责**开发、推理、代码调试**（无GPU，不做模型训练）；  
云GPU（AutoDL）：仅负责**模型训练、性能测试**（成本低，免费额度足够）。

## 二、具体分工
| 任务类型       | 执行环境       | 关键操作与工具                          | 注意事项                                  |
|----------------|----------------|-----------------------------------------|-------------------------------------------|
| 代码编写/调试  | 本地CPU        | VS Code、Python脚本、OpenCV预处理       | 推理时用YOLOv8n.pt（轻量化模型，CPU流畅） |
| 数据集标注     | 本地CPU        | LabelImg、OpenCV批量检查                | 标注文件存GitHub，同步到云GPU              |
| YOLOv8训练     | AutoDL云GPU    | RTX 3090实例、Ultralytics训练命令       | 训练后下载best.pt到本地，双渠道备份       |
| 模型推理测试   | 本地CPU        | ONNX Runtime、CPU量化优化               | 避免用大模型（如YOLOv8x.pt），防止卡顿    |
| 昇腾部署（后期）| 华为云免费实例 | ATC模型转换、AscendCL推理               | 本地仅做代码开发，部署在云平台验证        |

## 三、关键资源记录
1. AutoDL云GPU平台地址：https://www.autodl.com/（新用户免费额度≥100元）；  
2. 云GPU实例镜像选择：PyTorch 2.0.0 + Python 3.9（与本地环境一致）；  
3. 本地-云同步方式：GitHub仓库（代码/标注文件）+ 百度网盘（模型文件）。
